# 工程导向合成生物学零件数据库功能特点

## 数据库架构
### MongoDB数据模型
- 零件集合(parts)完整数据模型
  - 基本信息(ID、名称、描述、分类)
  - 来源信息(生物体、设计者、文献)
  - 序列信息(完整序列、核心序列、标准接口)
  - 工程特性(表达系统、组装特性、兼容性)
  - 实验数据(表达验证、功能测试、稳定性、安全性)
  - 使用统计(下载量、成功/失败案例、评分)
  - 元数据(创建/更新时间、版本)
- 优化的索引设计
  - 唯一索引: part_id
  - 常用查询索引: 名称、类别、载体、宿主等
  - 全文搜索: 序列
